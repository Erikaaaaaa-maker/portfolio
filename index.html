案例 01 · 车载记录仪

DR2850 性能旗舰记录仪体验设计

定义旗舰级车载记录仪的体验标准，从性能展示到用户操作的全链路体验把控。

项目背景

DR2850是70mai旗舰级行车记录仪产品，定位于对影像品质和产品体验有高要求的高端用户群体。项目需要在有限的硬件成本约束下，通过体验设计建立产品的差异化竞争力，实现"高端旗舰"的用户心智。

我的角色

作为该项目的核心体验设计师，我主导了从产品定义到设计交付的全链路体验设计工作。负责产品体验架构、交互设计、视觉规范、以及与硬件团队的跨端协作，同时推动设计规范的落地与验收。

核心挑战

挑战一

如何在有限的屏幕尺寸内，清晰展示旗舰级影像能力的差异？

挑战二

如何让用户感知到"高端"而非仅仅是"功能丰富"？

挑战三

设备端与App端的双端体验如何保持一致且互补？

挑战四

如何通过硬件反馈（音效、指示灯）强化品质感知？

关键策略 / 设计判断

"旗舰体验不是功能的堆砌，而是让用户在每个接触点都能感受到'这产品值这个价'。我们需要克制做加法，在关键节点做有感的体验放大。"

  • 采用"性能可视化"策略：通过动态UI和实时状态反馈，让用户直观感知产品能力
  • 建立"高端感"视觉语言：深色主题 + 精致动效 + 高对比度信息层次
  • 定义设备端与App端的体验边界：设备端做"快速查看"，App端做"深度管理"
  • 制定硬件反馈规范：指示灯状态、音效反馈与UI状态联动，强化品质感知

方案展开

[方案展开图区域 — 设备端UI / App界面 / 硬件反馈规范]
  • 设备端：重新设计主界面信息架构，突出关键状态（录制状态、影像规格、存储状态），次要信息通过手势或点击展开
  • App端：设计"驾驶档案"概念，让用户感受到产品的持续守护价值，而不仅仅是"工具"属性
  • 硬件反馈：制定指示灯多级亮度规范，设计开机关机的"仪式感"音效

推进与落地

  • 主导设计评审，与产品、硬件团队充分对齐，确保设计方案的技术可行性
  • 建立设计验收清单，推动设计还原度的逐项检查
  • 与固件团队协作，确保设备端动效在低算力设备上的流畅表现
  • 推动硬件团队落实指示灯亮度规范，实现UI与硬件反馈的联动

结果与影响

体验一致性

设备端与App端体验一致性达到95%以上，用户调研满意度显著提升

高端感知

用户对"高端感"和"品质感"的评分相比上一代产品提升明显

设计效率

基于该项目的设计规范向后续产品线迁移，效率提升约40%

验收闭环

建立的设计验收机制被推广至多条产品线，成为设计质量保障的标准流程

复盘总结

DR2850项目让我深刻理解"旗舰体验"的本质不是做加法，而是做取舍和聚焦。设计师需要有足够的判断力，在众多需求中找到最能打动目标用户的体验节点，并通过克制的设计语言将其放大。这个项目也验证了"设备端+App端+硬件反馈"三位一体体验设计方法论的有效性。

案例 02 · 车载后视镜

车载后视镜体验设计

创新形态产品的体验定义，解决多视角融合与用户操控安全的体验对齐问题。

项目背景

车载后视镜是70mai在形态创新上的重要尝试，将流媒体后视镜技术引入车载记录仪，实现前后双录与后视影像实时显示。产品定义了全新品类，但用户在"这是什么产品"和"怎么用"上存在认知断层。

我的角色

主导该创新品类从0到1的体验设计工作。核心挑战在于：用户对"后视镜"概念存在既有认知，操控复杂度高，而车载场景对安全性要求极高——体验设计必须在"易理解"与"功能强"之间找到平衡。

核心挑战

挑战一

用户心智教育：流媒体后视镜与传统后视镜有何不同？

挑战二

操控复杂度：多视角切换如何做到直觉且安全？

挑战三

显示延迟：后视画面延迟如何控制在安全范围内？

挑战四

成像品质：如何通过UI让用户感知后视画面与原镜的差异？

关键策略 / 设计判断

"创新品类的体验设计，第一步不是教用户怎么用，而是让用户先感受到'这和我有关'。我们需要把复杂的后视能力转化为用户能理解的体验语言。"

  • 设计"场景化引导"而非"功能说明书"：通过真实使用场景展示后视价值
  • 建立"单手操控"的直觉交互：触控区域与视角切换方向一致，降低学习成本
  • 设计"实时预览"体验：用户切换视角时实时看到画面，降低认知成本
  • 建立"后视模式 vs 普通模式"的对比可视化，让品质差异可感知

方案展开

[方案展开图区域 — 后视镜主界面 / 手势交互示意 / 多视角切换界面]
  • 设计"手势滑动"交互：触控区域与视角运动方向一致，降低学习成本
  • 建立"场景模式"概念：前视、后视、分屏等，预设视角与参数
  • 设计"后视就绪状态"的视觉反馈：通过动效和音效暗示设备正在工作
  • 实时画面与视角位置的同步反馈，让用户始终知道"屏幕在显示什么"

推进与落地

  • 与产品团队共创"场景化"产品定义，从功能导向转向用户任务导向
  • 推动固件团队实现"低延迟实时显示"，确保显示体验的流畅性
  • 设计用户测试：验证"手势滑动"的操控直觉性，针对性优化触控灵敏度
  • 建立后视镜产品的设计验收标准，包括显示延迟、反馈及时性、画面同步性等

结果与影响

用户理解度

产品上市后用户调研显示，"了解后视镜用途"的用户比例显著高于预期

功能使用率

多视角切换功能的使用率超过预期，用户愿意主动探索后视能力

操控满意度

后视镜操控满意度评分达到4.5/5.0

方法论沉淀

该项目的"创新品类体验设计方法"被沉淀为设计团队的方法论资产

复盘总结

车载后视镜项目教会我：创新品类的体验设计，需要先解决"认知"再解决"操作"。用户不会为一个陌生的功能付出学习成本，除非他能先感受到这和他有关。在设计上，我们采用了"先感受价值，再学习操作"的策略——通过场景化的价值展示让用户产生兴趣，再通过直觉的交互设计让用户快速上手。

案例 03 · 宠物智能饮水机

宠物智能饮水机体验设计

关注精致人宠生活，更关注爱宠健康，用科技为用户提供更健康&现代化的养宠方式。

项目背景

宠物智能饮水机是70mai切入宠物赛道的首款产品，旨在为宠物主人提供智能化、健康化的饮水解决方案。产品核心卖点是"活水循环"——通过循环流动的水吸引宠物多喝水，预防泌尿系统疾病。设计挑战在于：如何让主人感知到"这比普通水碗更值得买"，同时让宠物愿意主动使用。

我的角色

主导宠物智能饮水机的完整体验设计，包括产品定位、外观设计、App交互、情感化体验。核心挑战是定义"宠物智能饮水机"这一品类的体验标准，建立健康监测与主人关怀的平衡。

核心挑战

挑战一

健康可视化：如何让主人感知宠物喝水量是否充足？

挑战二

设备状态感知：饮水机滤芯、水量、清洁周期如何直观展示？

挑战三

宠物行为适配：如何让不同性格的宠物都愿意使用？

小米智能双摄门铃体验设计

小米智能双摄门铃创新性地采用双摄像头设计，实现门前全景覆盖。但双摄像头的画面如何呈现、用户如何在两个视角间切换、实时查看和历史回放如何设计——这些都没有先例可循。
